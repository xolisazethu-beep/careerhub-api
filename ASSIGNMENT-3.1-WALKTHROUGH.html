1. What I built

  • Toasts (sonner) for mutations — closing a job and submitting the wizard fire toasts; field errors stay inline.
  • Three-step ApplicationWizard on /jobs/[id] (alongside the existing apply panel): Details → Application → Review, one Zod schema, per-step trigger(), localStorage draft auto-save + restore, role gating.
  • AlertDialog confirmations for the two destructive actions: closing a listing and discarding a draft.
  • Skeletons & two distinct empty states on /jobs.
  • Stretch A/B/C: cross-tab draft sync, animated step transitions, LinkedIn live preview.

4. Part 3 — The application wizard

One Zod schema covers all three steps, with a cross-step refine on the LinkedIn URL. Navigation validates only the current step via trigger() with an explicit field list:

const STEP_FIELDS = [
  ["fullName", "email", "phone"],        // step 1
  ["coverLetter", "linkedinUrl", "source"], // step 2
  [],                                     // step 3 (review only)
];
const valid = await trigger(STEP_FIELDS[step], { shouldFocus: true });
if (!valid) return;          // Next does not advance on invalid current step

Draft auto-save uses a form.watch() subscription (with cleanup), keyed to the job:

const sub = watch((values) => {
  localStorage.setItem(`careerhub-application-${jobId}`, JSON.stringify(values));
  setHasDraft(true);
});
return () => sub.unsubscribe();   // unsubscribe so it can't fire after unmount

5. Part 4 — AlertDialog confirmations

Closing a job is a Server Action. AlertDialogAction renders in a portal outside any form, so type="submit" there does nothing. The fix: control the dialog with state and call the action programmatically in a transition:

function handleConfirm() {
  const fd = new FormData();
  fd.set("jobId", jobId);
  startTransition(async () => {
    const result = await closeJobListing(null, fd);   // called, not submitted
    result?.status === "success"
      ? toast.success(`Closed "${result.jobTitle}"`)
      : toast.error(result?.message ?? "Couldn't close the listing.");
    setOpen(false);
  });
}

7. Stretch goals

  • A — Cross-tab sync: a window 'storage' listener updates the draft/banner when another tab edits the same key. The event fires only in other tabs of the origin, never the one that wrote the change.
  • B — Animated transitions: library-free CSS keyframes; a direction state picks slide-from-right (Next) vs slide-from-left (Back) on an overflow-hidden container.
  • C — LinkedIn preview: best-effort preview from the URL slug with an <img onError> fallback to initials (LinkedIn has no public image API).
